# Use Project Harness

Read `.harness/entry.md`, inspect actual Git and workspace state, and run `orchestrator context audit --json` when available. Never treat chat memory, task titles, or generated summaries as product truth. If the CLI is unavailable, follow `references/fallback.md`.
