# Use Project Harness

Read `.harness/entry.md`, inspect actual Git and workspace state, and run `orchestrator context audit --json` when available. Use `specs/` for product meaning and `contracts/` for obligations. Ask only material unanswered questions, use TDD, check semantic conflicts, and keep normal Git names free of AI markers.

If deterministic checks are unavailable, follow `references/fallback.md` and state the reduced coverage.
